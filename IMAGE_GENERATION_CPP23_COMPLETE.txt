WHY NO ONNX/LIBTORCH FOR STABLE DIFFUSION?

Problem: Stable Diffusion export to ONNX/TorchScript is complex:
  ✗ UNET export failed: "Expecting a type not f<class 'typing.Union'>"
  ✗ Text Encoder export failed: Same type issue
  ✗ VAE Decoder export failed: Same type issue
  
Reason:
  - SD models use dynamic control flow
  - Complex attention mechanisms
  - Multiple interconnected components
  - PyTorch 2.x torch.onnx.export has limitations
  
Solutions for C++ Stable Diffusion:
  1. Use optimum library (Hugging Face) for proper ONNX export
  2. Use existing ONNX models from Hugging Face Hub
  3. Use LibTorch with full PyTorch model (requires large deps)
  4. Use specialized SD C++ implementations (e.g., stable-diffusion.cpp)

WHAT ABOUT LLAMA.CPP?
